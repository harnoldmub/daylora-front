# Captures Reelles Daylora

Dépose ici de vraies captures du backoffice Daylora pour remplacer les visuels actuels de la landing.

Objectif :
- montrer le vrai produit
- rassurer
- vendre la simplicité et la qualité
- éviter les mockups trop artificiels

## Dossier cible

Dépose les fichiers ici :

`client/public/marketing-real`

## Règles de capture

Prends de vraies captures du backoffice Daylora, pas des images générées.

À respecter :
- interface réelle Daylora
- pas de données sensibles visibles
- pas d'emails, noms complets, téléphones ou montants privés réels
- si besoin, utilise un compte de démo propre
- privilégier un écran desktop propre
- utiliser un zoom lisible
- éviter les captures trop larges avec trop d'espace vide
- éviter les modales parasites, toasts, curseurs, dropdowns ouverts inutilement
- garder une interface claire, premium et crédible

Format conseillé :
- `png`
- largeur entre `1400px` et `2200px`
- ratio visuel proche de `4:3`, `16:10` ou `3:2`

## Checklist de captures

### 1. `howitworks-start.png`

Capture réelle de l'étape de création ou de configuration initiale.

À montrer :
- création guidée
- informations de base du mariage
- structure simple étape par étape

Ce qu'on doit comprendre :
- démarrer est simple
- le produit guide l'utilisateur

### 2. `howitworks-personalize.png`

Capture réelle d'un écran de personnalisation.

À montrer :
- template choisi
- couleurs
- typographies
- sections
- visuel ou aperçu du site si possible

Ce qu'on doit comprendre :
- le site est personnalisable
- le rendu est premium

### 3. `howitworks-invitations.png`

Capture réelle de la gestion des invitations ou RSVP.

À montrer :
- invités
- types d'invitation
- statuts RSVP
- actions d'envoi ou de suivi si visibles

Ce qu'on doit comprendre :
- inviter et suivre les réponses est simple

### 4. `howitworks-publish.png`

Capture réelle d'un écran de publication ou d'aperçu final.

À montrer :
- aperçu du site
- URL ou slug personnalisé si possible
- bouton ou état de publication

Ce qu'on doit comprendre :
- le site est prêt à être mis en ligne rapidement

### 5. `feature-templates.png`

Capture réelle centrée sur les templates.

À montrer :
- plusieurs templates ou variations
- aperçu visuel clair
- différence de styles si possible

Ce qu'on doit comprendre :
- Daylora propose une vraie qualité de design

### 6. `feature-guests.png`

Capture réelle de la gestion des invités.

À montrer :
- liste d'invités
- statuts
- taille de groupe
- table attribuée
- filtres ou recherche si disponible

Ce qu'on doit comprendre :
- Daylora remplace les tableaux dispersés
- la gestion invités est claire

### 7. `feature-invitations.png`

Capture réelle d'une invitation personnalisée ou de son édition.

À montrer :
- page d'invitation par invité ou aperçu
- QR code si visible
- programme adapté ou type d'invitation
- rendu mobile-friendly si possible

Ce qu'on doit comprendre :
- chaque invité peut vivre une expérience personnalisée

### 8. `feature-gifts.png`

Capture réelle de la liste cadeaux ou cagnotte.

À montrer :
- cadeaux
- liens ou visuels
- montants suggérés ou modes de contribution
- cohérence visuelle avec le reste du produit

Ce qu'on doit comprendre :
- cadeaux et cagnotte sont gérés au même endroit

### 9. `feature-checkin.png`

Capture réelle du check-in jour J.

À montrer :
- recherche invité
- validation d'arrivée
- table ou statut
- vue mobile ou tablette si possible

Ce qu'on doit comprendre :
- l'accueil le jour J est fluide

### 10. `feature-support.png`

Capture réelle du support intégré.

À montrer :
- centre d'aide
- mini assistant
- espace de contact ou conversation

Ce qu'on doit comprendre :
- l'utilisateur n'est pas seul pendant la configuration

## Conseils de cadrage

Pour chaque capture :
- couper les barres système inutiles si elles polluent l'image
- garder le coeur de l'interface au centre
- montrer le bénéfice visible, pas toute l'application
- préférer une capture nette d'un seul usage plutôt qu'un écran trop chargé
